## Admin — Authentication

Manage stateful sessions for admins and superadmins.

**Source Files:**
- Route: `src/routes/admin/auth.routes.ts`
- Controller: `src/controllers/admin/auth.controller.ts`
- Model: `src/models/adminModels/session.model.ts`

---

### `POST /api/admin/auth/login` — Login Admin

Creates a secure database session and returns an HTTP-only signed cookie.

**Request Body**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | `string` | ✅ | Admin's email |
| `password` | `string` | ✅ | Admin's password |

**Response `200 OK`** (Sets `sid` Signed Cookie)
```json
{
  "success": true,
  "message": "Login successful",
  "user": {
    "id": "665c3d4e5f6...",
    "email": "admin@example.com",
    "role": "admin",
    "businessName": "Acme Corp",
    "businessDomain": "acme.lms.com"
  }
}
```

---

### `GET /api/admin/auth/me` — Verify Authentication

Verifies the user's current session via the `sid` signed cookie. Protected by `isAuthenticated` middleware.

**Request Body**: None

**Response `200 OK`**
```json
{
  "authenticated": true,
  "user": {
    "id": "665c3d4e5f6...",
    "email": "admin@example.com",
    "role": "admin",
    "businessName": "Acme Corp",
    "businessDomain": "acme.lms.com"
  }
}
```

---

### `POST /api/admin/auth/logout` — Logout Admin

Revokes the active session from the database and clears the user's `sid` cookie.

**Request Body**: None

**Response `200 OK`** (Clears `sid` Signed Cookie)
```json
{
  "message": "Logout successful"
}
```

Manage multi-tenant organizations — create, list, update, and delete tenants.

---

### `POST /api/tenants` — Create Tenant

Register a new tenant organization.

**Request Body**
| Field | Type | Required | Validation | Description |
|-------|------|----------|------------|-------------|
| `tenantName` | `string` | ✅ | min 3, max 100 | Organization name |
| `tenantDomainName` | `string` | ✅ | min 3, max 100, **unique** | Domain alias (e.g. `acme.lms.com`) |
| `tenantEmail` | `string` | ✅ | valid email, **unique** | Admin email address |
| `tenantExpireDate` | `string` | ✅ | min 3, max 100 | Expiry date (ISO date string, e.g. `2027-01-01`) |
| `tenantStatus` | `string` | ✅ | min 3, max 100 | Status: `active`, `inactive`, or `expired` |

> **Note**: `tenantDbName` is **auto-generated by the backend** from `tenantName` (e.g. `"Greenwood Academy"` → `"db_greenwood_academy"`). It is NOT sent by the client.

**Example Request**
```json
{
  "tenantName": "Greenwood Academy",
  "tenantDomainName": "greenwood.lms.com",
  "tenantEmail": "admin@greenwood.edu",
  "tenantExpireDate": "2027-12-31",
  "tenantStatus": "active"
}
```

**Response `201 Created`**
```json
{
  "success": true,
  "message": "Tenant created successfully",
  "data": {
    "_id": "665a1b2c3d4e5f6a7b8c9d0e",
    "tenantName": "Greenwood Academy",
    "tenantDomainName": "greenwood.lms.com",
    "tenantEmail": "admin@greenwood.edu",
    "tenantDbName": "db_greenwood_academy",
    "tenantExpireDate": "2027-12-31T00:00:00.000Z",
    "tenantStatus": "active",
    "__v": 0
  }
}
```

**Error Responses**

| Status | Condition | Body |
|--------|-----------|------|
| `400` | Zod validation failed | `{ "success": false, "message": "Invalid tenant data", "errors": {...} }` |
| `409` | Duplicate domain/email/dbName | `{ "success": false, "message": "A tenant with this domain, email, or database name already exists." }` |
| `500` | Server error | `{ "success": false, "message": "Tenant not created" }` |

### `PATCH /api/tenants/:id` — Update Tenant

Update an existing tenant by MongoDB `_id`.

**URL Params**
| Param | Type | Description |
|-------|------|-------------|
| `id` | `string` | MongoDB ObjectId of the tenant |

**Request Body** — Same fields as `POST /api/tenants` (all fields required by Zod validation). `tenantDbName` is **immutable** and cannot be changed.

**Example Request** — `PATCH /api/tenants/665a1b2c3d4e5f6a7b8c9d0e`
```json
{
  "tenantName": "Greenwood Academy (Updated)",
  "tenantDomainName": "greenwood.lms.com",
  "tenantEmail": "newadmin@greenwood.edu",
  "tenantExpireDate": "2028-12-31",
  "tenantStatus": "active"
}
```

**Response `200 OK`**
```json
{
  "success": true,
  "message": "Tenant updated successfully",
  "data": { "...updated tenant object..." }
}
```

**Error Responses**

| Status | Condition | Body |
|--------|-----------|------|
| `400` | Zod validation failed | `{ "success": false, "message": "Invalid tenant data", "errors": {...} }` |
| `404` | Tenant not found | `{ "success": false, "message": "Tenant not found" }` |
| `500` | Server error | `{ "success": false, "message": "Tenant not updated" }` |

## Superadmin — Form Templates

Manage dynamic form templates (e.g., Student Registration) that are shared across all tenants. These are global configurations set by the Superadmin.

**Source Files:**
- Route: `src/routes/superAdmin/formTemplate.routes.ts`
- Controller: `src/controllers/superAdmin/formTemplate.controller.ts`
- Model: `src/models/superAdmin/formTemplate.model.ts`

---

### `GET /api/superadmin/forms/:formId` — Get Form Template

Retrieve a form template configuration by its identifier.

**URL Params**
| Param | Type | Description |
|-------|------|-------------|
| `formId` | `string` | Form identifier (e.g. `student_registration`) |

**Request Body**: None

**Response `200 OK`**
```json
{
  "success": true,
  "data": {
    "_id": "665b2c3d4e5f6a7b8c9d0e1f",
    "formId": "student_registration",
    "name": "Student Registration Form",
    "fields": [
      {
        "id": "firstName",
        "label": "First Name",
        "type": "text",
        "required": true
      },
      {
        "id": "email",
        "label": "Email Address",
        "type": "email",
        "required": true,
        "isActive": true
      },
      {
        "id": "course",
        "label": "Course",
        "type": "select",
        "required": false,
        "isActive": true,
        "options": ["Math", "Science", "History"]
      }
    ],
    "__v": 0
  }
}
```

**Field Type Enum**: `text` | `email` | `number` | `date` | `select` | `textarea` | `file`

**Error Responses**

| Status | Condition | Body |
|--------|-----------|------|
| `404` | Template not found | `{ "success": false, "message": "Form template not found" }` |
| `500` | Server error | `{ "success": false, "message": "Internal server error" }` |

---

### `PUT /api/superadmin/forms/:formId` — Save Form Template (Upsert)

Create or update a form template. If a template with the given `formId` exists, it is updated; otherwise a new one is created.

**URL Params**
| Param | Type | Description |
|-------|------|-------------|
| `formId` | `string` | Form identifier (e.g. `student_registration`) |

**Request Body**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | ✅ | Display name for the form |
| `fields` | `Array<FieldConfig>` | ✅ | Array of field configuration objects |

**FieldConfig Object**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | ✅ | Unique field identifier (used as DB key) |
| `label` | `string` | ✅ | Display label shown on the form |
| `type` | `string` | ✅ | One of: `text`, `email`, `number`, `date`, `select`, `textarea`, `file` |
| `required` | `boolean` | ✅ | Whether the field is mandatory |
| `isActive` | `boolean` | ✅ | Whether the field is active (default inside code is true) |
| `options` | `string[]` | ❌ | Required only when `type` is `select` |

**Example Request** — `PUT /api/superadmin/forms/student_registration`
```json
{
  "name": "Student Registration Form",
  "fields": [
    { "id": "firstName", "label": "First Name", "type": "text", "required": true, "isActive": true },
    { "id": "lastName", "label": "Last Name", "type": "text", "required": true, "isActive": true },
    { "id": "email", "label": "Email", "type": "email", "required": true, "isActive": true },
    { "id": "dob", "label": "Date of Birth", "type": "date", "required": false, "isActive": true },
    { "id": "course", "label": "Course", "type": "select", "required": true, "isActive": true, "options": ["Math", "Science", "History"] }
  ]
}
```

**Response `200 OK`**
```json
{
  "success": true,
  "data": { "...upserted form template object..." }
}
```

**Error Responses**

| Status | Condition | Body |
|--------|-----------|------|
| `500` | Server error | `{ "success": false, "message": "Internal server error" }` |

## Admin — Student Management

Manage student registrations within a specific tenant's database. All routes in this group require the `tenantResolver` middleware, which resolves the tenant from the request headers or domain.

> ⚠️ **Middleware**: `tenantResolver` — Resolves tenant context from request. Populates `req.tenant` and `req.tenantDb` on the request object. Without a valid tenant context, these endpoints will fail.

**Source Files:**
- Route: `src/routes/admin/student.routes.ts`
- Controller: `src/controllers/admin/student.controller.ts`
- Model: `src/models/adminModels/student.model.ts`
- Middleware: `src/middlewares/tenantResolver.ts`

**Required Headers** (for local development):
| Header | Type | Description |
|--------|------|-------------|
| `x-tenant-domain` | `string` | The domain of the tenant (e.g. `greenwood.lms.com`) |

---

### `POST /api/admin/students` — Register Student

Create a new student in the tenant's specific database.

**Request Body**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `dynamicData` | `object (Map)` | ✅ | Key-value pairs matching the form template field IDs |

**Example Request**
```json
{
  "dynamicData": {
    "firstName": "John",
    "lastName": "Doe",
    "email": "john@example.com",
    "dob": "2000-05-15",
    "course": "Math"
  }
}
```

**Response `201 Created`**
```json
{
  "success": true,
  "data": {
    "_id": "665c3d4e5f6a7b8c9d0e1f2a",
    "dynamicData": {
      "firstName": "John",
      "lastName": "Doe",
      "email": "john@example.com",
      "dob": "2000-05-15",
      "course": "Math"
    },
    "createdAt": "2026-03-07T10:00:00.000Z",
    "updatedAt": "2026-03-07T10:00:00.000Z",
    "__v": 0
  }
}
```

**Error Responses**

| Status | Condition | Body |
|--------|-----------|------|
| `500` | Missing tenant DB / server error | `{ "success": false, "message": "Tenant database connection missing" }` or `{ "success": false, "message": "Internal server error" }` |

## Data Models Reference

### Tenant Schema
```typescript
{
  tenantName:       String  // required
  tenantDomainName: String  // required, unique
  tenantEmail:      String  // required, unique
  tenantDbName:     String  // required, unique, AUTO-GENERATED by backend from tenantName
  tenantExpireDate: Date    // required
  tenantStatus:     String  // required ("active" | "inactive" | "expired")
}
```

### Form Template Schema
```typescript
{
  formId:  String          // e.g. "student_registration"
  name:    String          // e.g. "Student Registration Form"
  fields:  [{
    id:       String       // unique field key
    label:    String       // display label
    type:     String       // "text" | "email" | "number" | "date" | "select" | "textarea" | "file"
    required: Boolean
    isActive: Boolean
    options:  [String]     // only for type "select"
  }]
}
```

### Student Schema (Tenant-Specific)
```typescript
{
  dynamicData: Map<String, Mixed>  // key-value pairs matching form template
  createdAt:   Date                // auto-generated
  updatedAt:   Date                // auto-generated
}
```

## Error Response Convention

All API responses follow a consistent structure:

**Success**
```json
{
  "success": true,
  "data": { ... },
  "message": "Optional success message"
}
```

**Error**
```json
{
  "success": false,
  "message": "Human-readable error description",
  "errors": { ... }  // Optional: Zod validation errors
}
```
